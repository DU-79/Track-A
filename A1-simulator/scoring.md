## 2. 性能基线评分

所有性能分均相对于官方基线。基线提供：

```text
T_compile_baseline = 基线编译时间
T1_baseline        = 基线单核仿真时间
T4_baseline        = 基线 4 核仿真时间
S_baseline         = T1_baseline / T4_baseline
```

参赛者提交产生：

```text
T_compile_submit = 参赛者编译时间
T1_submit        = 参赛者单核仿真时间
T4_submit        = 参赛者 4 核仿真时间
```

注：编译时间取首次编译和增量编译时间的平均值。

**功能正确性是前提：若某用例的 F1/F2 未通过，则该用例的 P1/P2/P3 得分为 0。**

### P1/P2 基于时间的性能

P1（编译）和 P2（仿真）使用相同的评分函数：

**P1：**

```text
R_compile = T_compile_baseline / T_compile_submit
Q_P1 = TimeScore(R_compile)
```

**P2：**

```text
R_sim = T1_baseline / T1_submit
Q_P2 = TimeScore(R_sim)
```

**`TimeScore(R)` 定义：**

| R | 含义 | 得分系数 |
|--:|------|---------:|
| R < 0.5 | 比基线慢 2 倍以上 | 0 |
| 0.5 <= R < 1.0 | 比基线慢 | `1.2 * (R - 0.5)` |
| R = 1.0 | 与基线持平 | 0.6 |
| 1.0 < R < 2.0 | 比基线快 | `0.6 + 0.4 * (R - 1.0)` |
| R >= 2.0 | 比基线快 2 倍及以上 | 1.0 |

达到基线可获得 60% 的性能分；超过基线继续累加；达到基线 2 倍时满分。

### P3 多核加速比

P3 比较参赛者的 4 核有效加速比相对于基线的表现：

```text
S_submit_raw = T1_submit / T4_submit
S_submit_cap = T1_baseline / T4_submit
S_submit_eff = min(S_submit_raw, S_submit_cap)
R_multi = S_submit_eff / S_baseline
Q_P3 = RatioScore(R_multi)
```

**`RatioScore(R)` 定义：**

| R | 含义 | 得分系数 |
|--:|------|---------:|
| R < 0.5 | 显著低于基线 | 0 |
| 0.5 <= R < 1.0 | 低于基线 | `1.2 * (R - 0.5)` |
| R = 1.0 | 与基线持平 | 0.6 |
| 1.0 < R < 1.5 | 高于基线 | `0.6 + 0.8 * (R - 1.0)` |
| R >= 1.5 | 达到基线 1.5 倍及以上 | 1.0 |

**P3 附加约束：**

1. 4 核仿真输出必须与单核输出 cycle-identical；否则该用例 `Q_P3 = 0`。
2. 若同一用例 `Q_P2 < 0.3`，则 `Q_P3 = 0`。
3. 若同一用例 `Q_P2 < 0.6`，则 `Q_P3` 上限为 0.6。

### 性能分计算

每个性能用例的最终得分：

```text
PerfScore(case) =
  W_P1(case) * Q_P1(case) +
  W_P2(case) * Q_P2(case) +
  W_P3(case) * Q_P3(case)
```

其中 `W_P1/W_P2/W_P3` 来自用例权重表。非性能用例的 P1/P2/P3 权重为 0。
